## 1. 项目背景

**设备用途**：解决木材壁炉在刮风天气时的烟气倒灌问题。设备安装于室外窗侧，在壁炉使用时通过向烟囱管道内送风形成正压，避免烟气从壁炉门缝冒出。

**使用者**：用户本人，冬季使用壁炉期间运行。

---

## 2. 功能需求

### Must — 必须实现

| 编号 | 功能描述 | 验收标准 |
|------|----------|----------|
| F-01 | 风扇 PWM 无级调速 | 0-100% 转速调节，PWM 频率≥20KHz，最低有效占空比 10%（设置值 1%-9% 自动提升至 10%，0% 为停止） |
| F-02 | 风扇转速反馈读取 | 四线风扇转速信号采集，测量误差≤5%，更新频率≥1Hz |
| F-03 | 堵转/停转检测与保护 | 设定占空比≥10% 时，转速反馈为 0 持续≥1.5s（可配置 100ms-5000ms）立即切断输出并报警；任意控制源发送启动指令可触发恢复尝试（启动 1.5s 检测转速）；设备重启自动清除保护状态 |
| F-04 | 双按键本地控制 | 加速键/减速键短按加减档（0-4 档，不循环），同时长按 >5s 恢复出厂设置，操作响应≤200ms |
| F-05 | 红外遥控控制（学习模式） | 支持 NEC/SONY/RC5/RC6/SAMSUNG 等常见协议，通过 Web 页面触发学习模式，按下遥控器对应按键自动记录编码；支持加速、减速、停止、30min/1h/2h/4h/8h 定时快捷设置；同一红外码只能绑定一个命令，重复学习时页面必须提示已绑定命令并继续等待其他按键；支持单独清除已学习命令；响应≤300ms |
| F-06 | 定时运行功能 | Web 页面支持 30min/1h/2h/4h/8h 预设快捷档位，支持自定义任意秒数（最大 356400 秒/99 小时）；红外快捷键支持 30min/1h/2h/4h/8h；定时精度≤1min/24h，倒计时结束自动停止 |
| F-07 | Web 控制页面 | 查看转速、状态、定时剩余时间，进行转速控制和定时设置，页面加载≤1s，密码鉴权（默认 admin:admin123） |
| F-08 | Web Service API | 转速控制、定时设置、状态查询、日志查询等原子操作，密码鉴权，响应≤500ms |
| F-09 | 运行日志持久化 | 至少 100 条日志，包含时间戳、事件类型、详情，断电不丢失，查询响应≤200ms |
| F-10 | OTA 固件升级 | 局域网 Web 端 OTA，升级过程断电可自动恢复，成功率≥99% |
| F-11 | WiFi 配网 | AP 配网模式，配网成功率≥95%，超时≤2min |
| F-12 | 断电状态恢复 | 供电恢复后 10s 内自动恢复到断电前的运行状态（转速、定时剩余时间等参数），或保持停止（可配置） |
| F-15 | 软启动/软停止 | 转速变化时默认 1 秒平滑渐变，渐变时间可配置 0-10 秒，0 为禁用 |
| F-17 | 低功耗休眠管理 | 风扇停止且无操作达到配置的等待时间后进入 Modem Sleep 低功耗模式，保持 WiFi 连接和 Web 服务，支持网络/红外/按键唤醒 |
| F-18 | mDNS 主机名访问 | 默认使用 `esp-fan.local`，局域网内无需记忆 IP |
| F-19 | Web 参数配置页面 | 修改软启动时间、休眠时间、访问密码、红外学习、上电恢复策略等所有可配置项；红外学习区列出所有命令并展示已学习的协议号和红外码 |
| F-22 | BOOT 键重置 WiFi | 长按 BOOT 键（GPIO0）1 秒清除 WiFi 凭证，设备重启进入 AP 配网模式 |
| F-23 | LED 指示灯 | 板载 LED 支持 PWM 调光，0 档熄灭，1-4 档对应 25%/50%/75%/100% 亮度；状态优先级为故障快闪（5Hz）> WiFi 未连接慢闪（1Hz）> 档位亮度/熄灭；所有有效用户操作反馈闪 1 下，默认 200ms，可配置 0-2000ms，0 表示关闭操作反馈；操作反馈采用反相脉冲，当前亮时短暂熄灭、当前灭时短暂点亮；非法参数、配置无变化、边界重复加减档、未知红外码和学习超时不闪 |

| 编号 | 功能描述 | 验收标准 |

- 不提供移动端 App，仅支持局域网 Web 页面
- 不支持 Home Assistant 等智能家居平台原生对接
- 不支持远程云控制，仅局域网访问
- 不支持电池供电，仅 12V 直流外置电源
- 不包含外壳设计与生产
- 不支持温度检测与联动

## 4. 硬件约束

- **芯片**：ESP-12F（ESP8266），4MB Flash
- **GPIO**：可用 GPIO 约 10 个，已分配：PWM 输出、转速反馈、双按键（2 个 IO）、红外接收、板载 LED（GPIO2）
- **电源**：12V DC 外置电源，最大输出电流≥5A
- **工作环境**：温度 -20℃~60℃，防水防尘 IP54，防小雨飞溅

## 6. 边界条件

| 场景 | 处理方式 |
|------|----------|
| 设备重启时正在执行定时任务 | 从持久化存储恢复转速和剩余定时时间，继续执行，误差≤10 秒 |
| 三个控制方式同时发送不同指令 | 按接收时间先后，最后一次指令生效 |
| 堵转保护触发后用户发送启动指令 | 任意控制源均可触发恢复尝试：启动 1.5s，检测到转速则恢复，否则维持保护；重启自动清除 |
| WiFi 断连期间用户发送 Web 指令 | 操作失效，本地/红外控制不受影响；WiFi 恢复后自动同步状态 |
| 定时时间到同时用户发送调速指令 | 先执行定时停止，再执行新调速指令 |
| 软启动过程中收到新的调速指令 | 立即中断当前渐变，从当前转速向新目标渐变 |
| 定时过程中用户重新设置定时 | 新定时直接覆盖旧定时，老任务立即取消 |
| NTP 对时失败 | 自动重试最多 3 次，失败不影响其他功能，日志继续使用相对时间；下次 WiFi 重连或 1 小时后再次尝试 |
| 红外学习超时 | 学习模式 10 秒超时自动退出，不保存未学习的按键 |
| 上电恢复策略为 false | 上电后保持停止状态，不自动恢复 |
| 长按 BOOT 键重置 WiFi | 清除凭证后立即重启，进入 AP 配网模式，屏蔽其他所有输入 |
| 按键到达边界 | 0 档按减速键无反应，4 档按加速键无反应（不循环） |
